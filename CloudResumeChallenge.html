About This Site

Built as part of the Cloud Resume Challenge to demonstrate cloud deployment, automation, and a small serverless app.

Frontend & DNS

  • Static site on Amazon S3 served globally through CloudFront (HTTPS via ACM).
  • Public DNS with Route 53: apex ALIAS to CloudFront; optional www CNAME/redirect.
  • Cache-control headers + automatic CloudFront invalidation on deploy for zero-stale rollouts.

Serverless Backend

  • API Gateway → Lambda (Python) to record visits; data stored in DynamoDB.
  • CORS enabled for the site origin; structured JSON responses consumed by the client.
  • CloudWatch logging for troubleshooting and basic observability.

CI/CD & Security

  • GitHub Actions with OIDC to assume a deploy role — no long‑lived AWS keys.
  • Least‑privilege IAM: scoped policies for S3 sync and CloudFront invalidation only.
  • Trust policy restricted to the repo/workflow; environment variables used for config (no secrets in code).
  • S3 access and CloudFront enforce HTTPS; DNS + certs managed by ACM/Route 53.

UI Features

  • Interactive visitor map using Leaflet with marker clustering and heat overlay.
  • Recent visitor list with country flags and ISP icons; total visit count.
